# Koml v0.0.2 — distribution layer

*Released: 2026-06-04*

> Browse, download, verify, and run curated GGUFs with a single API. JVM desktop joins Android and iOS as a first-class engine target.

v0.0.2 turns Koml from "works if you put a GGUF on disk" into "works if you call `coordinator.downloader.download(model).collect { ... }`." The public `:core` API surface is preserved — this release adds three new modules underneath plus a JVM target on the engine, all behind the existing `LlmKit.initialize() → LlmCoordinator → LlmSession` flow.

## What's new

### Real model registry and downloader

`coordinator.registry.curated()` now returns five ungated models bundled into the artifact:

| ID | Size | License | Context | RAM |
|---|---|---|---|---|
| `smollm2-135m-instruct-q8` | ~145 MB | Apache-2.0 | 2048 | 256 MB |
| `smollm2-1.7b-instruct-q4km` | ~1.05 GB | Apache-2.0 | 2048 | 1.5 GB |
| `tinyllama-1.1b-chat-q4km` | ~668 MB | Apache-2.0 | 2048 | 1 GB |
| `phi-3-mini-4k-instruct-q4` | ~2.39 GB | MIT | 4096 | 3 GB |
| `qwen2.5-1.5b-instruct-q4km` | ~986 MB | Apache-2.0 | 32768 | 1.5 GB |

`coordinator.downloader.download(model)` returns a `Flow<DownloadState>` that streams `Progress` events (throttled to ~4 Hz), terminates with `Completed(handle)` or `Failed(KomlException)`, supports resume across process restarts via HTTP `Range`, verifies SHA-256 as bytes arrive, and atomic-renames the verified file into place. Same Flow shape on Android, iOS, and JVM desktop.

```kotlin
coordinator.downloader.download(model).collect { state ->
    when (state) {
        is DownloadState.Progress  -> ui.showProgress(state.bytesDownloaded, state.totalBytes)
        is DownloadState.Completed -> ui.enableLoadButton(state.handle)
        is DownloadState.Failed    -> ui.showError(state.error.message)
        DownloadState.Paused       -> ui.showPaused()
    }
}
```

### License-acceptance gate

`DefaultModelDownloader` checks `ModelInfo.license.requiresAcceptance` before opening the HTTP connection. If unset, it emits `Failed(KomlException.LicenseNotAcceptedException)` immediately. Record an acceptance:

```kotlin
coordinator.acceptLicense("phi-3-mini-4k-instruct-q4")
```

State persists across process restarts via empty marker files under `<storage-root>/licenses/`. All five v0.0.2 curated models are permissively licensed (no acceptance required), so the gate is dormant — but it's wired and tested for when gated models (Llama, Gemma) land in a later release.

### JVM desktop engine (macOS arm64 + x64)

`:engine-llama` gained a `jvm()` target. The same `koml_jni.cpp` source as Android compiles to a per-arch `libkoml-jni.dylib`, packaged into the JAR under `META-INF/native/<arch>/`, and `System.load()`d once per JVM process via a synchronized idempotent loader.

```kotlin
// On macOS arm64 or x64
implementation("io.github.12345debdut:engine-llama-jvm:0.0.2")
val coordinator = LlmKit.initialize()
// ...same API as Android and iOS
```

Linux and Windows JVM users build from source via `scripts/build-llama-jvm.sh` for now. CI-driven multi-platform JARs land in a later release.

### Compose Desktop sample app

```bash
./gradlew :samples-desktop:run
```

Material 3 SwiftUI-style picker over the curated registry, download with live progress bar, load + streaming chat with cancel, delete-on-demand. macOS only in v0.0.2.

### Platform-aware storage

New `:storage` module abstracts the filesystem root per platform:

- **Android**: `<context.filesDir>/koml/` — context captured automatically at process start via `androidx.startup` (no user setup required).
- **iOS**: `<NSDocumentDirectory>/koml/` — app-private container.
- **JVM**: `$KOML_HOME` if set, else `~/.koml/`.

All models live under `<root>/models/<id>.gguf`; in-progress downloads at `<root>/models/<id>.gguf.part`; license acceptances at `<root>/licenses/<id>.accepted`. Atomic rename guarantees that anything at `<id>.gguf` (without `.part`) has been SHA-verified.

## What's gone

- `StubRegistry` and `StubDownloader` — deleted. `LlmKit.initialize()` now returns a coordinator backed by real implementations.
- Two cosmetic warnings: cinterop commonization (resolves [known-issue #3](../known-issues.md)) and expect/actual Beta (resolves #4) are both silenced with no behavior change.

## What's not changed

- `:core` public API — no breaking changes. `LlmCoordinator` gained one default-impl method (`acceptLicense`); everything else is identical to v0.0.1.
- Phase 1 sample code patterns. The new download flow is an *addition* to the existing generate flow, not a replacement.

## Getting started

Adds two new one-time steps on top of v0.0.1's setup:

```bash
# JVM macOS static libs (~10 min)
./scripts/build-llama-jvm.sh

# Real SHA-256s for the curated manifest (~30 min, ~5 GB of downloads)
./scripts/refresh-manifest-shas.sh
# Paste the output into registry/src/commonMain/kotlin/dev/koml/registry/CuratedModels.kt
```

Then build any sample:
```bash
./gradlew :samples-android:installDebug    # Android
./gradlew :samples-desktop:run             # Compose Desktop
cd samples-ios && xcodegen && open KomlSample.xcodeproj   # iOS
```

## Known limitations (carried over from v0.0.1)

- **No Metal on iOS** — still disabled due to b5460 ggml-common embed bug. macOS JVM keeps Metal on (the bug is iOS-specific).
- **No chat templates** — `session.chat()` still throws. Use `session.generate()` with a manually-formatted prompt.
- **No HuggingFace Hub search** — `coordinator.registry.searchHuggingFace(...)` returns `emptyList()`.

Plus the v0.0.2-specific items:

- **Manifest SHA-256s are placeholders** in the shipped source — see [known-issues #8](../known-issues.md). Real-world downloads will fail integrity checks until `refresh-manifest-shas.sh` is run.
- **JVM is macOS-only** in this release — see [known-issues #7](../known-issues.md) for the Linux/Windows recipe.

## Breaking changes

None. The `LlmCoordinator.acceptLicense` addition has a default no-op implementation, so any code that implemented `LlmCoordinator` against v0.0.1 still compiles and links.
