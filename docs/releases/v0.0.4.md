# Koml v0.0.4 — first publishable release

*Released: 2026-06-05*

> The library hasn't changed. Everything around it has.

v0.0.4 is the first release that's actually shippable to Maven Central. No new runtime features — everything in this release is infrastructure, governance, and process. If you've been using Koml from a local `./gradlew publishToMavenLocal`, this is the version that lets you put `implementation("io.github.12345debdut:engine-llama:0.0.4")` into a real consumer build (once we cut the actual Central publish, see [Maven Central status](#maven-central-status) below).

## What's new

### Maven Central publishing pipeline

`./gradlew publishAggregationToCentralPortal` from a maintainer machine:
- Builds every publishable module (`:core`, `:storage`, `:download`, `:registry`, `:engine-llama`).
- Signs every artifact with the maintainer's GPG key.
- Uploads a bundle to <https://central.sonatype.com> in **USER_MANAGED** state (a human still clicks "Publish" — no surprises in the early releases).

Backed by `com.gradleup.nmcp` 0.1.5 + Gradle's built-in `maven-publish` + `signing`. The full POM is wired: license, developer, SCM, issue tracker, all the fields Central validates against.

Maintainer walkthrough: [`docs/PUBLISHING.md`](../PUBLISHING.md).

### GitHub Actions CI

Every PR (and every push to `main`) now runs:
- JVM build + `:download:jvmTest` + `:engine-llama:jvmTest` on **Ubuntu** and **macOS** in parallel.
- A separate Android-Kotlin compile job on Ubuntu (catches Kotlin/AGP regressions without needing pre-built `libkoml-jni.dylib` slices).

~5–10 min total. Test reports get uploaded as artifacts on failure for triage.

### Governance docs

- **`LICENSE`** — Apache 2.0.
- **`CODE_OF_CONDUCT.md`** — Contributor Covenant 2.1.
- **`CONTRIBUTING.md`** — dev setup, branching, conventional commits, what makes a good PR.
- **[`docs/VERSIONING.md`](../VERSIONING.md)** — defines the public API surface and the pre-1.0 SemVer carve-out:
  - `0.0.x` patch releases are **strictly backward-compatible** — bug fixes, docs, additive default-impl methods only.
  - `0.x.0` minor releases **may break** the public API. Read the release notes.
  - llama.cpp version bumps count as a minor.

### Structured issue + PR templates

The four templates under `.github/ISSUE_TEMPLATE/`:

- **Bug report** — what happened, repro, target (multi-select), version, environment, logs.
- **Feature request** — motivation, Kotlin sketch, alternatives, size dropdown, contribution offer.
- **Model request** — for the curated list. Required checkboxes enforce: ungated, permissive license, <4 GB, verified to load on llama.cpp b5460.
- **Discussions link** (via `config.yml`) for open-ended questions, so the Issues tab stays focused on actionable items.

Plus a PR template with the checklist you actually want reviewers to verify.

### Configuration cache enabled

`org.gradle.configuration-cache=true` is now on by default, after auditing the three highest-risk paths:
- JVM tests
- `:samples-android:assembleDebug` (under AGP 9 with the bypass)
- `:engine-llama:assembleKomlEngineDebugXCFramework` (SKIE + cinterop)

Roughly halves config time on warm runs. Resolves the known-issue #10 that has been outstanding since Phase 1.

## What hasn't changed

- Every public API in `:core`. v0.0.3 code keeps working.
- The five curated models, the chat templates, the cinterop bindings.
- Platform coverage: still Android, iOS, macOS JVM, macOS Kotlin/Native. Linux from-source.

## Maven Central status

The pipeline is wired and tested locally via `./gradlew publishToMavenLocal`. **The first actual publish to Central is still pending** — it requires the maintainer to:
1. Register the `dev.koml` namespace on the Central Portal.
2. Generate the GPG key and publish the public half to a keyserver.
3. Run `./gradlew publishAggregationToCentralPortal` and click Publish at central.sonatype.com.

When that lands, this section gets updated with the actual coordinates.
